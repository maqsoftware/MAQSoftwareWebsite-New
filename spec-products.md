# Products — Consolidated Spec

Single source of truth for the **Products** experience on the MAQ Software
marketing site. Combines navigation, listing, detail, illustration, impact
metrics, theming and content authoring rules.

> Status: **v2 (consolidated)** — covers the image illustration system and
> "Proven impact" stat cards added on top of v1.

---

## 1. Goals

- Help enterprise visitors discover MAQ Software's six accelerator products.
- Make each product feel **shippable**: real screenshot-like illustration,
  concrete impact numbers, clear next step.
- Stay inside the **"Editorial Red"** palette (one brand red + cool
  neutral system, white chrome). Inspired by Databricks / Hitachi /
  Insight: a single accent color carries the brand; neutrals carry the
  content. Never introduce navy/blue/teal/amber/sand accents.
- Keep all assets **inline** — no external image dependencies, no CDN, no
  binary check-ins beyond the existing favicon/icon set.
- Drive every CTA to `customersuccess@maqsoftware.com`.

---

## 2. Routes & navigation

| Route | Page | Purpose |
|---|---|---|
| `/products` | `ProductsHome` | Listing of all 6 products with thumbnails and aggregated impact strip. |
| `/products/:slug` | `ProductDetail` | Per-product page with hero illustration, capabilities, impact cards, use cases. |

- Header **Products** menu is a MegaMenu rendered from `products[]`.
  - Lists each product (icon + name + tagline).
  - Final row: "All products →" linking to `/products`.
- Unknown `:slug` → `<Navigate to="/products" replace />`.

## 3. Data model (`src/data/products.tsx`)

```ts
interface ProductFeature  { title: string; desc: string }
interface ProductUseCase  { industry: string; scenario: string }
interface ProductImpact   { value: string; label: string }

interface Product {
  slug: string;            // URL slug, e.g. "migratefast"
  initials: string;        // 2-letter logo monogram
  name: string;            // Display name
  tagline: string;         // 2–4 word category
  shortDesc: string;       // 1–2 sentence elevator pitch (listing card)
  longDesc: string;        // 2–4 sentence narrative (detail hero)
  heroIcon: ReactNode;     // Fluent icon (see §3a icon allow-list)
  image: string;           // data:image/svg+xml URI (see §3b)
  tags: string[];          // 2–4 short badges
  features: ProductFeature[];   // 3–5
  useCases: ProductUseCase[];   // 2–4
  platforms: string[];          // 2–5 Microsoft / Azure stack names
  impact: ProductImpact[];      // EXACTLY 3 (see §3c)
}
```

Helpers:

```ts
export const products: Product[];
export function getProduct(slug: string): Product | undefined;
```

### 3a. Icon allow-list

Use Fluent UI v9 `@fluentui/react-icons` only. Verified safe names used by
this catalog:

- `Bot24Regular`
- `Settings24Regular`
- `ArrowSwap24Regular`
- `Code24Regular`
- `ShieldCheckmark24Regular` &nbsp;*(NOT `CheckmarkSeal24Regular` — does not exist)*
- `CloudArrowUp24Regular`

### 3b. Illustration system (`image: string`)

Each product carries an inline **SVG data URI** generated by the
`illustration(motif)` helper. This keeps the bundle clean (no external
assets, CSP-friendly, cache-trivial) while still rendering as a real
`<img>` element.

**Palette (strict — no other colors allowed in any motif):**

| Token | Hex | Use |
|---|---|---|
| `RED` | `#c8102e` | Primary accent, brand mark |
| `RED_PALE` | `#fde7ea` | Soft fills, secondary bars |
| `CREAM` | `#f7f8fa` | Card header bar, source blocks (cool neutral) |
| `SLATE` | `#475467` | Secondary text, neutral accents |
| `BORDER` | `#e4e7ec` | Card chrome, dividers |
| `INK` | `#0f172a` | Primary text in illustration |

**Frame:** every illustration uses the same 560×360 viewBox with a white
"window" card, a cream titlebar, and three traffic-light dots
(red / red-pale / border). Body changes per motif.

**Motif catalog:**

| Product | Motif | Visual story |
|---|---|---|
| `ai-datalens` | `chat` | Q/A bubbles ending in a red "Ask" send button |
| `fabric-admin-agent` | `gauge` | 98% health donut + utilization bars |
| `migratefast` | `swap` | LEGACY card → red exchange icon → FABRIC card |
| `embedfast` | `embed` | Tabbed dashboard with bar chart + donut |
| `certyfast` | `shield` | Shield + checkmark + CERTIFIED stamp panel |
| `loadfast` | `ingest` | 3 sources → LoadFAST → Bronze/Silver/Gold medallion |

### 3c. Impact authoring rules

Every product **must** ship with exactly **3** `impact` entries. Each entry
is one *headline metric* + one *outcome label*.

- `value`: short, scannable, concrete. Allowed forms:
  - Percentage: `"85%"`, `"60%"`
  - Count: `"120+"`, `"500+"`, `"30+"`
  - Time: `"5 days"`, `"5 min"`, `"24/7"`
  - Multiplier: `"10×"`, `"3×"`
  - Currency: `"$4M+"`
  - SLO/ratio: `"99.95%"`, `"100%"`, `"0"`
- `label`: 3–8 words. Outcome phrasing (`"Faster certification turnaround"`),
  NOT feature phrasing (`"Has certification gates"`).
- Avoid "up to" — pick the number we actually quote in pursuits.

| Token | Value | Where |
|---|---|---|
| `--maq-red` | `#c8102e` | Primary accent, impact `value` text, buttons, link |
| `--maq-red-dark` | `#9b0c24` | Hover / pressed states |
| `--maq-red-pale` | `#fde7ea` | Card borders on cool surfaces, soft bars |
| `--maq-red-50` | `#fff5f6` | Pale-red surface, used sparingly |
| `--maq-surface-cream` | `#f7f8fa` | **Cool neutral** — detail visual frame, impact cards, listing thumb bg (NOT warm sand anymore) |
| `--maq-surface-soft` | `#f0f2f5` | Slightly deeper neutral block |
| `--maq-off-white` | `#f7f8fa` | Section background |
| `--maq-black` | `#0f172a` | Slate 900 — H1 / H2 |
| `--maq-ink` | `#1f2937` | H3 / strong body |
| `--maq-gray-700` | `#475467` | Body copy |
| `--maq-gray-500` | `#667085` | Muted / labels |
| `--maq-border` | `#e4e7ec` | Cool neutral dividers |
| `--maq-border-strong` | `#cdd2da` | Emphasized dividers |
| `--maq-shadow-sm/md` | cool shadow | Card hover lift |

**Forbidden:** any navy / royal blue / teal / cyan / amber / warm-sand
surface. The palette is intentionally one brand red + a cool neutral
ladder — the same pattern Databricks, Hitachi Solutions, Insight and
Avanade all use. If a new component needs depth, use the neutral surface
ladder (`--maq-off-white` → `--maq-surface-soft`) with red used only as
a 1–2% accent.

**Typography:** body and headings use **Inter** (loaded via Google Fonts
`@import`) with `font-feature-settings: "cv02","cv03","cv04","cv11"` for
refined letterforms. Global rhythm: `h1 { line-height: 1.1; letter-spacing: -0.02em }`,
`h2 { line-height: 1.15; letter-spacing: -0.015em }`, `p { line-height: 1.6 }`.

## 9. Accessibility

- `<img>` for illustrations carries a descriptive `alt`. (We do NOT mark
  the hero as `aria-hidden` — the alt provides meaning.)
- Decorative icon tile under the image uses `aria-hidden="true"`.
- Impact cards are inside a `<section aria-label="Outcomes">` so screen
  readers announce them as a region.
- All interactive controls are real Fluent `<Button>` / `<Link>` — no
  click-bait `<div>`s.
- Color contrast: red on cream and red on white both clear WCAG AA at
  the sizes used (≥38px for `value`, ≥13px for labels).

---

## 10. Performance

- Illustrations are inline SVG data URIs → 0 extra HTTP requests, ~1–3 KB
  each gzipped inside JS bundle. `loading="lazy"` on listing thumbnails
  for safety (cheap with data URIs but consistent with future raster
  swaps).
- Single client bundle (`dist/assets/index-*.js`) currently ~521 KB raw /
  ~156 KB gzip. Acceptable; revisit code-splitting only if it crosses
  ~600 KB gzip.

## 11. Adding a new product — checklist

1. Pick a slug (kebab-case, URL-safe).
2. Pick a Fluent icon from the allow-list (§3a) — verify it exists.
3. Pick or add a motif. To add a motif, extend the `illustration()` switch
   using ONLY the §3b palette and the standard 560×360 chrome.
4. Write `shortDesc` (1–2 sentences) and `longDesc` (2–4 sentences).
5. Provide 3–5 `features`, 2–4 `useCases`, 2–5 `platforms`.
6. Provide **exactly 3** `impact` entries following §3c rules.
7. Pick 2–4 `tags` (max 24 chars each).
8. Add the entry to the `products` array — order in the array is the
   order users see in the MegaMenu AND on `/products`.
9. `npm run build` — must exit 0 with no TypeScript errors.

No other code changes are required: the header menu, listing grid, and
detail page all derive automatically from `products[]`.

## 13. Shareable single-file build

Goal: hand a single file to a stakeholder (PM, sales, exec) and have
them see the **exact same UI** without installing Node, running a
server, or hosting anything.

### 13.1 Artifact

- Path: `share/maqsoftware.html`
- Format: self-contained HTML5 document with **all** CSS and JS
  inlined (`<style>...</style>` + `<script type="module">...</script>`).
- Size: ~512 KB (uncompressed). Compresses to roughly 150 KB if zipped.
- Dependencies: none. No external CDN calls, no asset folder, no
  server. SVG illustrations are already inline data URIs in the bundle.

### 13.2 How to share

- Email or chat the single `maqsoftware.html` file.
- Recipient double-clicks the file → opens in their default browser
  (`file://`) and the full site renders, including routing.
- Works offline. Works behind corporate firewalls. Works on any OS.

### 13.3 Build settings that make this possible

- `vite.config.ts` → `base: "./"` so generated asset URLs are relative
  (required for `file://`).
- `src/main.tsx` uses `HashRouter` from `react-router-dom`, not
  `BrowserRouter`. This makes routes look like
  `maqsoftware.html#/products/migratefast` and removes any dependency
  on a server-side rewrite for SPA routing.

### 13.4 How to regenerate

Run from repo root (PowerShell):

```powershell
npm run build

$dist = Join-Path $PWD 'dist'
$html = Get-Content (Join-Path $dist 'index.html') -Raw

# Inline CSS
$html = [regex]::Replace($html, '<link\s+rel="stylesheet"[^>]*href="([^"]+)"[^>]*/?>', {
  param($m)
  $href = $m.Groups[1].Value.TrimStart('/').TrimStart('.').TrimStart('/')
  $p = Join-Path $dist $href
  if (Test-Path $p) { return "<style>$(Get-Content $p -Raw)</style>" } else { return $m.Value }
})

# Inline JS module
$html = [regex]::Replace($html, '<script\s+type="module"\s+crossorigin\s+src="([^"]+)"\s*></script>', {
  param($m)
  $src = $m.Groups[1].Value.TrimStart('/').TrimStart('.').TrimStart('/')
  $p = Join-Path $dist $src
  if (Test-Path $p) { return "<script type=`"module`">$(Get-Content $p -Raw)</script>" } else { return $m.Value }
})

New-Item -ItemType Directory -Force -Path 'share' | Out-Null
Set-Content -Path 'share\maqsoftware.html' -Value $html -Encoding UTF8
```

### 13.5 Constraints

- The shareable file is a **snapshot**. Re-run the build after any
  content/code change and re-share the file.
- Browser `mailto:` links work from `file://` and will open the
  user's default mail client.
- Do NOT inline images that are not already data URIs — keep the
  inline-SVG illustration policy from §3b. Raster assets would either
  break the single-file promise or require base64 inlining (rejected
  for size).
